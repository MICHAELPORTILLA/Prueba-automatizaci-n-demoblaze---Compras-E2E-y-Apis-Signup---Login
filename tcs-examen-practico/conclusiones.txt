1. Cobertura Completa de Pruebas de Apis

Se validaron 5 escenarios críticos de la API de DemoBlaze:

- Creación exitosa de usuario (@signup-success).

- Intento de crear usuario existente (@signup-fail).

- Login con usuario inexistente (@login-user-not-exist).

- Login con contraseña incorrecta (@login-Wrong-password).

- Login exitoso (@login-success).


2. Manejo Correcto de Casos de Error:

La API responde adecuadamente con mensajes de error específicos:

- "User does not exist." - Usuario no encontrado.

- "Wrong password." - Contraseña incorrecta.

- "This user already exist." - Usuario duplicado.

Todas las validaciones de mensajes de error pasaron exitosamente.


3. Buenas Prácticas de Automatización Implementadas:

- Generación dinámica de datos: Uso de función randomString() para crear usuarios únicos.

- Manejo de contraseñas: Passwords en Base64 (cGFzc3dvcmQ=) para cumplir con el formato requerido.

- Estructura organizada: Separación clara entre escenarios de Signup y Login usando tags


4. Validaciones Exitosas:

- Registro exitoso: Status 200 sin validación específica del response (vacio/String).

- Login exitoso: Validación de que el response contiene el token de autenticación "Auth_token:".

- Login con credenciales incorrectas: Validación exacta de mensajes de error en formato JSON.


5. Rendimiento de Pruebas:

- Tiempo total de ejecución: ~5.62 segundos para los 5 escenarios.

- Promedio por escenario: ~1.12 segundos, lo cual es eficiente para pruebas de API.

- Mayor tiempo: Primer escenario (0.67s) - posiblemente por inicialización.


6. Resultado General:

- 100% de pruebas exitosas (5/5 escenarios pasados)

- 0 fallos en toda la suite de pruebas

- Cobertura completa de los flows principales de autenticación de DemoBlaze
